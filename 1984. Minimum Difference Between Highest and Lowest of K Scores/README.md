Пример: nums = \[9,4,1,7] k = 3 Output 5

- [9,4,1] -> 8
- [9,4,7] -> 5
- [9,1,7] -> 8
- [4,1,7] -> 6

Алгоритм: 
1. Отсортируем исходный массив nums
2. рассмотрим префикс длины k : [nums[0], ..., nums[k-1]]. Заметим, что для idx > k-1 нет смысла анализировать nums[0]. Действительно, выполняется инвариант: nums[k-1] - nums[0] <= nums[idx] - nums[0]. Поэтому искомое для nums[0] равно nums[k-1] - nums[0], иные элементы массива нет смысла рассматривать, так как дадут не меньший результат.
3. рассмотрим [nums[1], ..., nums[k]]. На прошлом шаге выяснили, что для nums[k] не теряется элементов (в данном случае речь о nums[0]), которых могли бы оптимизировать ответ. Для nums[1] рассуждения аналогичные предыдущему пункту
4. рассмотрим [nums[i], ..., nums[k + i - 1]]. Тут уже действуют индуктивыне рассуждения для [nums[0], ..., nums[i-1]] и [nums[k + i], ..., nums[len(nums)] - 1] аналогичные пункту 2 и 3
